# Initialize Central Learnings Repository

## Context
Create a central personal repository to store all your learning patterns. This becomes your single source of truth for accumulated wisdom across all projects.

## Input
- Path to central repo (default: `~/.pi/learnings-central`)
- Optional: Git remote URL for backup

## Steps

### 1. Validate Path

```bash
CENTRAL_PATH="${1:-$HOME/.pi/learnings-central}"

# Expand ~ to full path
CENTRAL_PATH="${CENTRAL_PATH/#\~/$HOME}"

if [ -d "$CENTRAL_PATH" ]; then
  echo "⚠️  Central repo already exists at: $CENTRAL_PATH"
  echo ""
  echo "Options:"
  echo "  1. Use existing repo (recommended)"
  echo "  2. Reinitialize (will backup existing)"
  echo "  3. Choose different path"
  echo ""
  read -p "Choice (1/2/3): " choice

  case $choice in
    2) mv "$CENTRAL_PATH" "${CENTRAL_PATH}.backup.$(date +%Y%m%d_%H%M%S)" ;;
    3) read -p "Enter new path: " CENTRAL_PATH ;;
    *) echo "Using existing repo"; exit 0 ;;
  esac
fi
```

### 2. Create Directory Structure

```bash
mkdir -p "$CENTRAL_PATH"
cd "$CENTRAL_PATH"

# Create subdirectories
mkdir -p patterns

# Initialize git
git init
```

### 3. Create Initial Catalog

```bash
cat > catalog.yaml <<'EOF'
# Personal Learning Patterns Catalog
# Central repository for accumulated wisdom across all projects

metadata:
  owner: $(whoami)
  created: $(date -I)
  last_updated: $(date -I)
  description: "Personal learning patterns from all projects"

# Auto-promotion threshold
promotion_threshold: 3

# Central patterns (accumulated from all projects)
patterns: []

# Statistics
stats:
  total_patterns: 0
  total_occurrences: 0
  projects_contributing: []
EOF
```

### 4. Create README

```bash
cat > README.md <<'EOF'
# Personal Learning Patterns

This is my central repository for learning patterns accumulated across all projects.

## What This Is

- **Single source of truth** for my accumulated development wisdom
- **Pattern catalog** from all projects I work on
- **Shareable** across projects via `/learnings sync`
- **Git-backed** for version history and backup

## Structure

```
.
├── catalog.yaml          # Master catalog of all patterns
├── patterns/             # Pattern markdown files
│   ├── missing-returns.md
│   ├── forgot-team-scope.md
│   └── ...
└── README.md
```

## Usage

### In any project:

```bash
# Pull patterns from central
/learnings sync

# Share pattern to central
/learnings push <pattern-key>

# View central catalog
cat ~/.pi/learnings-central/catalog.yaml
```

## Backup

This repo should be backed up to GitHub (private repo recommended):

```bash
cd ~/.pi/learnings-central
git remote add origin git@github.com:yourname/pi-learnings-private.git
git push -u origin main
```

## Pattern Lifecycle

1. Pattern discovered in project → logged locally
2. Pattern promoted (3 occurrences) → applied locally
3. Pattern validated (works well) → pushed to central
4. New project → syncs from central → gets accumulated wisdom

## Maintenance

- Patterns are added via `/learnings push` from projects
- Catalog automatically updated on push
- Git history tracks evolution of patterns
- Can browse/edit patterns directly in this repo

---

Generated by pi_launchpad learning system
EOF
```

### 5. Create .gitignore

```bash
cat > .gitignore <<'EOF'

# Temp files

tmp/

```

### 7. Configure Pi Launchpad

Update `~/.pi/config.sh` (create if doesn't exist):

```bash
CONFIG_FILE="$HOME/.pi/config.sh"
mkdir -p "$HOME/.pi"

# Add or update LEARNINGS_CENTRAL variable
if [ -f "$CONFIG_FILE" ]; then
  # Check if already configured
  if grep -q "PI_LEARNINGS_CENTRAL" "$CONFIG_FILE"; then
    echo "ℹ️  Central repo already configured in ~/.pi/config.sh"
  else
    # Append configuration
    cat >> "$CONFIG_FILE" <<EOF

# Central learnings repository
export PI_LEARNINGS_CENTRAL="$CENTRAL_PATH"
export PI_LEARNINGS_AUTO_SYNC="true"      # Auto-sync on session start
export PI_LEARNINGS_AUTO_PUSH="prompt"    # Prompt before pushing
EOF
  fi
else
  # Create new config file
  cat > "$CONFIG_FILE" <<EOF
#!/bin/bash
# Pi Launchpad Configuration

# Central learnings repository
export PI_LEARNINGS_CENTRAL="$CENTRAL_PATH"
export PI_LEARNINGS_AUTO_SYNC="true"      # Auto-sync on session start
export PI_LEARNINGS_AUTO_PUSH="prompt"    # Prompt before pushing

# Load project-specific config if exists
if [ -f "\$PWD/.pi/config.sh" ]; then
  source "\$PWD/.pi/config.sh"
fi
EOF
  chmod +x "$CONFIG_FILE"
fi
```

### 8. Optional: Set Up Git Remote

```bash
echo ""
echo "📦 Central repo initialized!"
echo ""
echo "Optional: Set up GitHub backup"
echo ""
read -p "Create GitHub remote? (y/n): " setup_remote

if [ "$setup_remote" = "y" ]; then
  echo ""
  echo "Create a new private repo on GitHub:"
  echo "  https://github.com/new"
  echo ""
  read -p "Enter repo URL (git@github.com:user/repo.git): " remote_url

  if [ -n "$remote_url" ]; then
    git remote add origin "$remote_url"

    echo ""
    read -p "Push to GitHub now? (y/n): " do_push

    if [ "$do_push" = "y" ]; then
      git push -u origin main
      echo "✅ Pushed to GitHub"
    else
      echo "ℹ️  Push later with: cd $CENTRAL_PATH && git push -u origin main"
    fi
  fi
fi
```

### 9. Report Success

```bash
echo ""
echo "✅ Central learnings repository initialized"
echo ""
echo "Location: $CENTRAL_PATH"
echo "Catalog:  $CENTRAL_PATH/catalog.yaml"
echo "Patterns: $CENTRAL_PATH/patterns/"
echo ""
echo "Configuration updated:"
echo "  ~/.pi/config.sh"
echo ""
echo "Next steps:"
echo ""
echo "1. In any project, push a pattern:"
echo "   /learnings push <pattern-key>"
echo ""
echo "2. In a new project, sync from central:"
echo "   /learnings sync"
echo ""
echo "3. View central catalog:"
echo "   cat $CENTRAL_PATH/catalog.yaml"

# Show git status if remote configured
if git remote get-url origin >/dev/null 2>&1; then
  REMOTE_URL=$(git remote get-url origin)
  echo "GitHub remote: $REMOTE_URL"
  echo ""
fi
```

## Error Handling

**Permission denied:**
```
Error: Cannot create directory at /path/to/central

Check:
- Path is writable
- Parent directories exist
- No permission restrictions
```

**Git not installed:**
```

Install git:
  brew install git   (macOS)
  apt install git    (Linux)
```

```

  2. Update config to new path

Choice:
```

## Advanced Usage

### Multiple Central Repos

```bash
# Personal patterns
/learnings init-central ~/.pi/learnings-personal

# Work patterns (separate repo)
/learnings init-central ~/work/learnings-work
```

Then configure per-project which central to use:

```bash
# In project/.pi/config.sh
export PI_LEARNINGS_CENTRAL="$HOME/work/learnings-work"
```

---

### Import Existing Patterns

If you already have patterns in a project:

```bash
# Initialize central
/learnings init-central

# From existing project
cd ~/projects/existing-project
/learnings push --all-promoted

# All promoted patterns now in central
```

## Maintenance

### Backup Central Repo

```bash
# Manual backup
cd ~/.pi/learnings-central
tar -czf ~/backups/learnings-$(date +%Y%m%d).tar.gz .

# Or use git remote (recommended)
git push origin main
```

### View History

```bash
cd ~/.pi/learnings-central

# See pattern evolution
git log --oneline patterns/

# See specific pattern history
git log --follow patterns/missing-returns.md
```

### Clean Up

```bash
# Remove patterns that are no longer useful
cd ~/.pi/learnings-central
git rm patterns/obsolete-pattern.md
git commit -m "chore: remove obsolete pattern"

# Update catalog to remove references
```
